CASES21 Export Kit

How to get your student roster out of CASES21 and into Lumi — for first-time setup and for the annual class rollover. Version 1.1.

What you'll do

  1. Open the CASES21 SQL worksheet

    Follow this menu path:

    Utilities → MAZE → View and Report Data → Worksheet → SQL editor

  2. Run the Lumi query

    Paste in the whole of lumi-cases21-export-kit.sql (downloaded from the same place as this guide) and run it. You should see one row per currently-enrolled student, sorted by year level then home group.

  3. Export the results to Excel

    Use the worksheet's export to Microsoft Excel option. Nothing needs editing afterwards — please don't rename or reorder the columns.

  4. Upload it to Lumi

    In Lumi go to Students, then:

    • Setting up for the first time — choose Import CSV
    • Moving to a new school year — choose School Year Transition

    Upload the Excel file exactly as it was saved. Lumi shows you what it read before anything is saved, and a year rollover can be undone afterwards.

Three ways to hand Lumi the data — all equally fine.

Upload the .xlsx straight from Excel; or Save As → CSV (Comma delimited) and upload that; or select the rows in Excel, copy, and use Lumi's Paste from Excel box. Use whichever is easiest.

Column order doesn't matter and extra columns are ignored. The uppercase headings the worksheet produces are read correctly.

Doing the annual rollover rather than first-time setup?

Parent emails aren't needed — Lumi already has them. The SQL file ends with a five-column variant that leaves contact details out of the export entirely.

If something looks wrong

Lumi says it can't find First Name, Last Name or Class Name
The heading row is missing. Re-export including headings, or when pasting, start the selection at the heading row.
Some students show "Missing required fields"
Those students have no home group in CASES21. Assign one there and re-export, or exclude those rows in Lumi's review step.
A student appears twice, or Lumi refuses a duplicate ID
Two CASES21 records share one student key. Lumi won't guess between them — correct it in CASES21 and re-export.
Lots of students are flagged as having left the school
The export is probably incomplete — one campus or one year level only. Check the row count against your enrolment before confirming. For a large number, Lumi asks you to tick an extra confirmation.
Next year's Prep students are missing
They're in CASES21 with a future enrolment status. The SQL file has a commented option that includes them — swap the WHERE line as shown.
The query returns an error mentioning CAST
Your CASES21 version handles the year-level conversion differently. The SQL file's header comment shows a one-line replacement; Lumi reads that form too.
Lumi won't accept the file
Lumi reads .xlsx, not the older .xls. In Excel choose File → Save As → Excel Workbook (.xlsx).

Before you send the file anywhere

This export contains student names and, for first-time setup, family email addresses. Upload it to Lumi directly rather than emailing it, and delete the file from your computer once the import is confirmed.
